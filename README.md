## 📝 Описание

Данный проект проверяет корректную генерацию виджета через цепочку **конфигурация → применение → генерация превью → отображение**.  
Используется **паттерн Builder**, чтобы абстрагировать тест от низкоуровневых деталей и легко расширять сценарии.

---

## 🛠 Технологии

- **TypeScript** — язык тестов  
- **Playwright** — фреймворк для E2E-тестирования  
- **Allure** — генерация отчётов  
- Node.js, npm — управление зависимостями  


## 🚀 Установка и запуск

Для локального запуска потребуется клонировать репозиторий и прописать зависимости.

1. git clone https://github.com/lumediluna/task
2. cd task
3. npm install -D @playwright/test allure-playwright allure-commandline
4. npm run test:report


## 🔍 Описание теста: Генерация виджета

Тест: Генерация виджета
Этот автотест проверяет, что виджет корректно генерируется и отображается на странице при использовании валидной конфигурации, собранной через паттерн Builder.
Что именно делает тест:
Готовит конфигурацию виджета
Через WidgetBuilder создаётся конфигурация с преднастроенными параметрами (структура, данные, внешний вид и т.п.), абстрагируя тест от “низкоуровневых” деталей и позволяя легко расширять сценарии.
Применяет конфигурацию на странице виджета
Фикстура widgetPage инкапсулирует работу со страницей: методом applyConfig(config) конфигурация передаётся в систему, эмулируя реальные действия пользователя или интеграции.
Генерирует превью виджета
Шаг generatePreview() нажимает кнопку “Сгенерировать превью” и запускает процесс отрисовки виджета с текущими настройками, как это произойдёт у реального пользователя.
Проверяет успешное отображение
expectWidgetVisible() убеждается, что виджет реально появился на странице и доступен пользователю.
takeScreenshot() сохраняет визуальное состояние виджета для последующего анализа (визуальная регрессия, отладка падений, документация поведения).
Ценность теста:
Гарантирует, что цепочка «конфигурация → применение → генерация превью → отображение» работает end‑to‑end.
Делает сценарий легко расширяемым: можно добавлять новые варианты конфигураций в WidgetBuilder без переписывания тестовой логики.
Обеспечивает как функциональную проверку (видимость виджета), так и визуальный контроль (скриншоты).
